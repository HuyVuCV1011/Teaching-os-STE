# Agent Instructions for `docs/`

Public docs describe product intent, setup, architecture, and safe workflows.

- Keep docs public-safe: no real student data, private prompts, credentials, internal benchmarks, or confidential school information.
- Preserve the docs-first, schema-first style: concise, practical, and aligned with current implementation status.
- Do not document features as complete unless the code supports them.
- Keep grading, review, audit, versioning, and data-safety language consistent across docs.
- Link to existing docs instead of duplicating large sections.

Report whether documentation-only changes were made or code behavior changed too.
